# Investment ROI and CAGR Analytics Design

## Project Overview
The current Sure app calculates investment returns using a simple unrealized gains formula: `(Current Value - Cost Basis) / Cost Basis`. This method is inaccurate for portfolios with continuous deposits and withdrawals because it ignores cash flow timing and realized gains.

This project will build a robust analytics engine and UI to accurately track investment performance, specifically:
- Total Net Cash Invested
- Total Gains (including realized and reinvested income)
- Personal ROI / Internal Rate of Return (IRR)
- Benchmark Return / Time-Weighted Return (TWR)

## 1. Calculation Engine (Backend)

We will introduce a new service/model, `InvestmentPerformance`, which takes a `Family` (or subset of investment accounts) and a `Period`.

### A. Net Cash Invested
- **Definition**: The true out-of-pocket cash put into the portfolio.
- **Calculation**: Sum of all `Entry` records marked as transfers/deposits into the investment accounts, minus withdrawals.

### B. Personal ROI (Internal Rate of Return - IRR)
- **Definition**: Money-Weighted Return. Evaluates the performance of the portfolio accounting for the exact timing and size of the user's deposits and withdrawals.
- **Calculation**: 
  - We will implement the XIRR (Extended Internal Rate of Return) algorithm.
  - **Inputs**: Array of cash flows. Positive values for deposits, negative for withdrawals. The final cash flow is the current portfolio value (represented as a negative withdrawal today).
  - **Algorithm**: A Newton-Raphson approximation to find the rate `r` where Net Present Value (NPV) = 0.

### C. Benchmark Return (Time-Weighted Return - TWR)
- **Definition**: Evaluates the pure performance of the underlying assets, eliminating the skew caused by the user's cash flow timing.
- **Calculation**: 
  - Use daily `Holding` snapshots combined with the daily `Entry` cash flows.
  - Break the overall period into sub-periods divided by cash flow days.
  - Sub-period return = `(Ending Value - Cash Flow) / Starting Value`
  - Chain the sub-period returns: `[(1 + R1) * (1 + R2) * ... ] - 1`
  - Annualize the result for multi-year periods.

### D. Reinvested Gains
- **Definition**: Income generated by the portfolio (dividends and interest).
- **Calculation**: Leverage the existing `Totals` system to sum `total_dividends` and `total_interest`.

## 2. Testing Strategy (TDD)
- Comprehensive test coverage for `InvestmentPerformance`.
- **Test Cases for XIRR**:
  - Single deposit with growth.
  - Multiple deposits with varying growth.
  - Negative returns.
  - Edge cases: Zero cash flows, flat portfolio, single-day holding period.
- **Test Cases for TWR**:
  - Validate math against known spreadsheet examples.
  - Ensure missing historical prices don't crash the calculation.
- **Existing Tests Check**:
  - The current `Trend` and `unrealized_gains_trend` methods do have tests (`test/models/holding_test.rb` and `test/models/investment_statement_test.rb`). We will verify them to ensure they aren't introducing false negatives before deploying the new engine.

## 3. User Interface

We will update the `app/views/investments/index.html.erb` dashboard to feature a new top-level **Performance Overview** card.

### Key Metrics to Display
1. **Net Cash Invested**: The baseline amount.
2. **Current Portfolio Value**: The current total.
3. **Total Gains**: Dollar difference between Current Value and Net Cash Invested.
4. **Income**: Total Dividends/Interest earned.
5. **Personal Return (IRR)**: Prominent percentage representing the user's true annualized yield.
6. **Benchmark Return (TWR)**: Secondary percentage for comparing against market indices.
